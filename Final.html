Scheme 1: Recommend Highest Rated Movies

The first scheme is based on the highest average ratings of movies within each genre. To compute this we take a number of steps below.

movieAvgRat  = colMeans(as(Rmat, 'matrix'), na.rm = TRUE) 

genre_highly_rated_matrix = matrix(NA, 18, length(unique(ratings$MovieID)))
rownames(genre_highly_rated_matrix) = genre_list

Rmat_Cols  = strtoi(strip(colnames(Rmat), char="m"))

for(i in 1:18) {
  
  tmp = data.frame(id = movies$MovieID[genre_matrix[,i] == 1], rat = 1:length(movies$MovieID[genre_matrix[,i] == 1]))
  tmp[,2] =NA
  
  tmp2 = movieAvgRat[Rmat_Cols %in% movies$MovieID[genre_matrix[,i] == 1]] 
  
  for(j in tmp$id){
    if(!is.na(j)){
      tmp[which(tmp$id == j),2] = tmp2[paste0('m',j)]
    }
  }
  
  tmp = tmp[order(tmp$rat,decreasing = TRUE),]
  for(k in 1:dim(tmp)[1]){
    if(!is.na(tmp[k,2])){
      genre_highly_rated_matrix[i,k] = tmp[k,1]
    }
  }
}

First we iterate to sort all movies into the 18 genres. After, we are getting a list of the movies for a specific genre, then sorting the movies within the genre based on movies that received the highest average rating from Movie Lens users. This is then stored in a matrix for our shiny app to access.

Scheme 2: Recommend Most Popular Movies

Our second scheme follows the first scheme closely algorithm wise, with a couple of key differences as we are trying to rank the movies with each genre based on popularity rather than highest average rating.

temp = as(Rmat, 'matrix')
temp[which(temp < 3)] = NA
temp[which(temp >= 3)] = 1

moviePopCount = colSums(temp, na.rm = TRUE)  

genre_most_popular_matrix = matrix(NA, 18, length(unique(ratings$MovieID)))
rownames(genre_most_popular_matrix) = genre_list

for(i in 1:18) {
  
  tmp = data.frame(id = movies$MovieID[genre_matrix[,i] == 1], rat = 1:length(movies$MovieID[genre_matrix[,i] == 1]))
  tmp[,2] =NA
  
  tmp2 = moviePopCount[Rmat_Cols %in% movies$MovieID[genre_matrix[,i] == 1]] 
  
  for(j in tmp$id){
    if(!is.na(j)){
      tmp[which(tmp$id == j),2] = tmp2[paste0('m',j)]
    }
  }
  
  tmp = tmp[order(tmp$rat,decreasing = TRUE),]
  for(k in 1:dim(tmp)[1]){
    if(!is.na(tmp[k,2])){
      genre_most_popular_matrix[i,k] = tmp[k,1]
    }
  }
}

To accomplish sorting by popularity we employ a method that ranks the movies with the highest quantity of >3 star ratings highest. Then similarly to scheme 1 we are applying this to all 18 genres and returning a matrix that our shiny app is able to access.

Evaluating User-Based and Item-Based Collaborative Filtering

We opted to use RMSE, MSE, and MAE as our performance metrics. We followed the same evaluation methods and metrics for both UBCF and IBCF.

We also decided to normalize our rating matrix to remove any rating bias that may have been introduced by the users. This solves 3 problems, first is the human tendency to have bias towards certain ratings for either good or bad movies. This also solves the issue of different users having different scales for rating movies, and lastly also forces our distribution closer to a normal one, which will assist when running our evaluation metrics at the end.

e <- evaluationScheme(Rmat, method="split", train=0.9, given=10, goodRating=3)

ubcf <- Recommender(getData(e, "train"), "UBCF")

ibcf <- Recommender(getData(e, "train"), "IBCF")

ubcf_pred <- predict(ubcf, getData(e, "known"), type="ratings")

Comparing top-N recommendations (Evaluation)

We did derive metrics for MAE but not WMAE as we felt the weighting was fair as is. Additionally we leverage RMSE primarily as our performance metric.

In our case we did have “NA” values present in our final matrix. We opted to sort based on rating value and then filter out null values upon injection to our front end system.

We chose a nearest neighbor size of 50 given the large size of our dataset, and high percentage of sparsity. This in addition to relatively homogeneous ratings gives us the ability to use a larger number of nearest neighbors for clustering. In addition to this we used cosine as our similarity metric.

scheme <- evaluationScheme(Rmat, method="split", train = .9, k=10, given=-5, goodRating=3)

algorithms <- list("user-based CF" = list(name="UBCF", param=list(nn=50)),  "item-based CF" = list(name="IBCF", param=list(k=50)))

results <- evaluate(scheme, algorithms, type = "topNList", n=c(1, 3, 5, 10, 15, 20))

Ultimately we had much lower error when applying user based collaborative filtering so we chose to use UBCF for our shiny app. This is based on the lower evaluation metrics, namely RMSE. Also oure true positive rate compared to false positive rate was much more desirable for the user based cf. Lastly, we see higher precision even at higher sensitivities for the user based cf which further affirms usage of our user based cf model.
